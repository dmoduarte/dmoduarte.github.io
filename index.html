A Model-Driven Approach for Mobile Business Intelligence

MSc dissertation, Out 2015 to Dec 2016 Faculdade de Ciências e Tecnologias UNL with the collaboration of Novabase

This work addresses the problem of developing a hybrid mobile solution towards the Business Intelligence (BI) domain, offering monitoring services and simultaneously addressing the problem of user empowerment, with easy configuration and semi-automatic generation of analytical widgets (mainly charts). To provide such feature to the user, from a theoretical and conceptual perspective the proposed prototype is based on the design of a Domain Specific Modeling Language (DSML), aligned with the Model-Driven Development (MDD) approach and inspired by the Product Lines (SPL) principles.

We built a modular system composed by an editor, a server and a mobile BI application. The editor is responsible for encompassing the DSML concepts by providing the end user with the tools to configure analytical widgets, the Server is responsible to implement the MDD concepts by managing the transformation process, which reads the configuration made on the Editor by the end user and generates the widget. The result of such process is then interpreted and displayed by the mobile BI application. These componenents were integrated with the implementation of a REST web-service.

File Server Replication and Synchronization

Group project of Distributed Systems subject: Mar 2015 to May 2015, Faculdade de Ciências e Tecnologias UNL

In this project me and my colleague developed a cloud-like system wherein we had clients, contact and file servers as main components. The goal of the client component is to send files to the file server component. The file server component could be phisically replicated, then it would synchronize with its replicas. Among the file server, its replicas and the client, there is a contact server which is used as: An interface component between the client and the file server, based on a command-line interface; A master component responsible for naming, managing the file servers, initialize the replication and synchronization processes.

We used the Java Remote Method Invocation (Java RMI) as interface of comunication between the components. Lastly, we included the Dropbox services in our file server group by using their REST API and a proxy also developed by us. Every time the client would upload a file to Dropbox, using the contact server CLI, our primary file server and its replicas would synchronize with the Dropbox services.

Computer Added Design of Multidimensional Models

Project of Data Warehousing subject: May 2015 to Jun 2015, Faculdade de Ciências e Tecnologias UNL

Based upon the ideas of the article : "From Enterprise Models to Dimensional Models: A Methodology for Data Warehouse and Data Mart Design" - from Daniel L. Moody", the goal of this system is to support the user on the derivation of transactional models (OLTP models) to analytical models (OLAP models), used to store historical information for analytical purposes, e.g. Reporting.

The prototype uses a MySQL JDBC driver to connect with a MySQL database to extract the metadata from a 3NF relational schema. Then, based on the concepts from the article it heuristically classifies and processes the entities to come up with a multidimensional model. At any time the user can interact with the model, which is graphically presented with the JGraph - a Java graph library.

Business Intelligence solution for Energy and Utilities

Project taken during the bachelor's curricular internship: Abr 2014 to Jul 2014, Novabase

In this project me and my colleague developed a Business Intelligence (BI) solution, under the supervision of our team leader for an utility client. The final goal was to have a set of reports that displays the corporate information for analytical contexts given a set of information (real) generated from energy equipments.

The BI solution is a layered architecture which is responsible for periodically extract operational data from operational sources, transform it and load it (ETL) into analytical Data Warehouse. The data that resides there is then presented in reports.

The ETL processes were developed and exported using Talend Open Source. The Data Warehouse was implemented with PostgreSQL and the reports designed and implemented with SAP Business Objects WEBI. The SAP reports were then integrated into a product from Novabase using a SAP BO SDK.
