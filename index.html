A Model-Driven Approach for Mobile Business Intelligence

MSc dissertation, Out 2015 to Dec 2016 Faculdade de Ciencias e Tecnologias UNL with the collaboration of Novabase

This work addresses the problem of developing a hybrid mobile solution towards the Business Intelligence (BI) domain, offering monitoring services and simultaneously addressing the problem of user empowerment, with easy configuration and semi-automatic generation of analytical widgets (mainly charts). To provide such feature to the end user, from a theoretical and conceptual perspective the proposed prototype is based on the design of a Domain Specific Modeling Language (DSML), aligned with the Model-Driven Development (MDD) approach and inspired by the Product Lines (SPL) principles.

I built a modular system composed by an editor, a server and a mobile BI application. The editor is responsible for encompassing the DSML concepts by providing the end user with the tools to configure analytical widgets, the Server is responsible to implement the MDD concepts by managing the transformation process, which reads the configuration made on the Editor by the end user and generates the widget. The result of such process is then interpreted and displayed by the mobile BI application. These components were integrated with the implementation of a REST web-service.

Vacaciones

Group project of Internet Applications Devevelopment subject: Dec 2015, Faculdade de Ciencias e Tecnologias UNL

In this group project, me and my group colleagues developed an hotel booking prototype where we focused, mainly, on the views, web services and access management. There was different types of user groups which provided different accesses on the web site: Administrators, Hotel Administrators, Review Moderators and Clients.

  • A Client is responsible for making and managing his own reservations, review the hotels
  • A Review Moderator is responsible for managing the reviews made by the clients on a specific hotel
  • An Hotel Administrator is responsible for registrating and managing hotels and all its reservations, managing the Review Moderators
  • An Administrator is responsible for accepting new hotel registrations, managing the registered hotels and its administrators

We used Angular JS, JQuery, HTML5 and CSS3 to develop the front-end and the Spring Framework and Hibernate as backend tools. We used Spring Security to implement the access model.

Mobile BI Data Server

Out 2015, Novabase

In this same period, along with my participation in the development of a hybrid mobile application for the energy sector, called 'Mobile Business Intelligence' - offering a set of dashboards for monitoring purposes -, I developed a prototype data server whose goal is to synchronize with the mobile client by providing it with data stored in remote database. All the work I've done in this stage served as basis for the dissertation's prototype.

The data server was developed with Node.JS and encompassing the Express.JS framework. The remote data was stored in a mockup database implemented with PostgreSQL.

File Server Replication and Synchronization

Group project of Distributed Systems subject: Mar 2015 to May 2015, Faculdade de Ciencias e Tecnologias UNL

In this project me and my group colleague developed a cloud-like system wherein we had clients, contact and file servers as main components. The goal of the client component is to send files to the file server component. The file server component could be phisically replicated, then it would synchronize with its replicas. Among the file server, its replicas and the client, there is a contact server which is used as: An interface component between the client and the file server, based on a command-line interface; A master component responsible for naming, managing the file servers, initialize the replication and synchronization processes.

We used the Java Remote Method Invocation (Java RMI) as interface of comunication between the components. Lastly, we included the Dropbox services in our file server group by using their REST API and a proxy also developed by us. Every time the client would upload a file to Dropbox, using the contact server CLI, our primary file server and its replicas would synchronize with the Dropbox services.

Computer Added Design of Multidimensional Models

Project of Data Warehousing subject: May 2015 to Jun 2015, Faculdade de Ciencias e Tecnologias UNL

Based upon the ideas of the article : "From Enterprise Models to Dimensional Models: A Methodology for Data Warehouse and Data Mart Design" - from Daniel L. Moody", the goal of this system is to support the user on the derivation of transactional models (OLTP models) to analytical models (OLAP models), used to store historical information for analytical purposes, e.g. Reporting.

The prototype uses a MySQL JDBC driver to connect with a MySQL database to extract the metadata from a 3NF relational schema. Then, based on the concepts from the article it heuristically classifies and processes the entities to come up with a multidimensional model. At any time the user can interact with the model, which is graphically presented with the JGraph - a Java graph library.
